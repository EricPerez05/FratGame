# Build A Club Feature Board

## Purpose

This board tracks the current implementation tracks for the team.
Each contributor should claim one feature area and update progress in this file.

## Sprint Focus

- Fine tune build mode
- UI updates
- Stat implementation
- Logo generation

## How To Use

1. Add your name under the Owner column for one feature.
2. Keep Status updated as work progresses.
3. Add short notes and links to relevant commits or PRs.
4. Move completed items to the Completed section.

- Not Started: Work has not begun.

- Blocked: Waiting on dependency, decision, or fix.
- Review: Work complete, waiting for review/testing.
- Complete: Approved and merged.

## Implementation Notes

### Fine tune build mode

- Scope:
  - Basement detection reliability
  - Build-mode enter/exit behavior
  - PropGrid visibility behavior
  - Teleport blocking while in build mode
- Risks:
  - Desync between client and server state
  - Edge-case basement detection overlap

### UI updates

- Scope:
  - Build panel polish
  - Button hierarchy and readability
  - Mobile-safe spacing and scaling
- Risks:
  - Conflicts with existing game HUD scripts

### Stat implementation

- Scope:
  - Define stats schema
  - Sync leaderstats with backend data
  - Wire stat changes into gameplay loops
- Risks:
  - Data migration and persistence errors

### Logo generation

- Scope:
  - Style exploration
  - Final logo export set
  - Roblox-ready asset sizing
- Risks:
  - Brand direction changes late in sprint
